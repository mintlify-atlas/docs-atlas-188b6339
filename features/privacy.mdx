## Privacy-First Architecture

LuminTime is designed from the ground up to protect your privacy. All data collection, storage, and analysis happens entirely on your local device. There are no servers, no accounts, and no data transmission.

<Note>
**Core Privacy Principles:**
- 100% local-first: All data stays in your browser's IndexedDB
- No telemetry: We don't collect usage statistics or crash reports
- No cloud sync: Your data is never transmitted anywhere
- No accounts: No sign-up, login, or authentication required
- Open source: All code is publicly auditable
</Note>

## Local-First Data Storage

### IndexedDB Storage

All browsing activity data is stored in your browser's IndexedDB, a local database that:

- Resides entirely on your device
- Is isolated per browser profile
- Cannot be accessed by websites you visit
- Is only accessible by the extension itself

```typescript
// Database initialization (from db/index.ts)
import Dexie from 'dexie';

export const db = new Dexie('LuminTimeDB');

db.version(1).stores({
  history: '++id, date, [date+hostname], hostname, startTime',
  sites: '[date+hostname], date, hostname',
  pages: '[date+hostname+path], date, hostname',
  meta: 'key',
  hourlyStats: '[date+hour], date'
});
```

### No Network Requests

LuminTime never makes network requests. The extension:

- Doesn't communicate with external servers
- Doesn't send analytics or telemetry
- Doesn't download updates or sync data
- Doesn't use third-party services or APIs

<Info>
You can verify this by checking the extension's network activity in your browser's developer tools. You'll see zero requests.
</Info>

### Not Tracked

- Browser internal pages (chrome://, about:, etc.)
- Page content or text
- Form inputs or passwords
- Cookies or local storage from websites
- Mouse movements or keyboard input
- Screenshots or visual data
- Your IP address or location

### Trackable URLs Only

Only http and https URLs are tracked:

```typescript
function isTrackableWebUrl(url: string): boolean {
  if (!url) return false;
  
  try {
    const parsedUrl = new URL(url);
    return parsedUrl.protocol === 'http:' || parsedUrl.protocol === 'https:';
  } catch {
    return false;
  }
}
```

<Warning>
Browser internal pages like `chrome://settings`, `about:config`, and extension pages are never tracked.
</Warning>

## Data Isolation

### Browser Profile Isolation

Each browser profile has its own isolated LuminTime database:

- Work profile data is separate from personal profile
- Guest/incognito windows use separate storage
- Data cannot be shared between profiles

### Extension Permissions

LuminTime requires minimal browser permissions:

```json
// From manifest (wxt.config.ts)
{
  "permissions": [
    "tabs",           // Read active tab URLs and titles
    "idle",           // Detect when user goes idle
    "alarms",         // Periodic session saves
    "storage",        // Save active session state
    "webNavigation"   // Detect page navigation
  ],
  "host_permissions": [
    "<all_urls>"      // Access tab URLs (not page content)
  ]
}
```

<Note>
While LuminTime requests `<all_urls>` permission, this is only used to read tab URLs from the Tabs API. The extension cannot and does not access page content, cookies, or other sensitive data.
</Note>

## Data Retention

To manage storage space, old raw history logs are aggregated:

- **Raw logs**: Kept for 7 days by default (configurable: 7, 14, 30, or 90 days)
- **Hourly summaries**: Created from raw logs older than retention period
- **Daily aggregates**: Never deleted automatically

```typescript
// Retention job runs every hour
export async function runRetentionJob(maxDays = 3): Promise<void> {
  const rawRetentionDays = await getRawRetentionDays();
  const cutoff = formatDate(addDays(parseDate(getTodayStr()), -(rawRetentionDays - 1)));
  
  for (let i = 0; i < maxDays; i++) {
    const oldest = await db.history.orderBy('date').first();
    if (!oldest || oldest.date >= cutoff) break;
    await aggregateOneDay(oldest.date);
  }
}
```

<Note>
Aggregation preserves hourly time data while reducing storage usage. Your daily and weekly analytics remain accurate even after raw logs are aggregated.
</Note>

## Security Considerations

### Browser Security Model

LuminTime benefits from the browser's security model:

- Extension code runs in an isolated context
- IndexedDB is encrypted at rest by the OS (when device encryption is enabled)
- Data is protected by the browser's sandbox
- Only the extension can access its own storage

### No Authentication Vulnerabilities

With no accounts or servers, entire classes of vulnerabilities are eliminated:

- No password breaches
- No session hijacking
- No CSRF attacks
- No SQL injection
- No API key leaks
- No man-in-the-middle attacks

<Warning>
If someone gains physical or remote access to your device and browser profile, they could access your LuminTime data. Use device encryption and strong browser profile passwords to protect against this.
</Warning>
