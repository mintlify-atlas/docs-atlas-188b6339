## Overview

LuminTime uses **WXT** to build optimized extension packages for multiple browsers. The build process:

- Bundles and minifies all code
- Optimizes assets (icons, images)
- Generates browser-specific manifests
- Creates distribution-ready ZIP files

## Build Commands

<CodeGroup>

```bash Production Build
bun run build
# Builds for all target browsers
# Output: .output/chrome-mv3, .output/firefox-mv2
# Minified, optimized, ready for distribution
```

```bash Create ZIP Packages
bun run zip
# Builds AND creates .zip files
# Output: .output/lumin-time-1.0.0-chrome.zip
#         .output/lumin-time-1.0.0-firefox.zip
```

```bash Firefox-Specific ZIP
bun run zip:firefox
# Creates Firefox-only package
# Uses -b firefox flag
# Output: .output/lumin-time-1.0.0-firefox.zip
```

</CodeGroup>

## Build Process

<Steps>
  <Step title="Run Build Command">
    ```bash
    bun run build
    ```
    
    WXT compiles TypeScript, bundles dependencies, and optimizes assets.
  </Step>
  
  <Step title="Check Build Output">
    ```bash
    ls -la .output/
    ```
    
    You should see:
    ```
    .output/
    ├── chrome-mv3/         # Chrome/Edge/Brave build
    │   ├── manifest.json
    │   ├── background.js   # Bundled background worker
    │   ├── popup.html
    │   ├── assets/         # Compiled JS/CSS
    │   └── icons/          # Extension icons
    └── firefox-mv2/        # Firefox build
        ├── manifest.json
        ├── background.js
        └── ...
    ```
  </Step>
  
  <Step title="Create Distribution Package">
    ```bash
    bun run zip
    ```
    
    Creates ZIP files ready for store submission.
  </Step>
  
  <Step title="Verify Package">
    Test the built extension before publishing:
    
    **Chrome:**
    1. Navigate to `chrome://extensions`
    2. Enable "Developer mode"
    3. Click "Load unpacked"
    4. Select `.output/chrome-mv3` folder
    
    **Firefox:**
    1. Navigate to `about:debugging#/runtime/this-firefox`
    2. Click "Load Temporary Add-on"
    3. Select `.output/firefox-mv2/manifest.json`
  </Step>
</Steps>

## Build Configuration

### WXT Config

From `wxt.config.ts`:

```typescript
import { defineConfig } from "wxt";

export default defineConfig({
  srcDir: "./src",
  modules: [
    "@wxt-dev/module-vue",      // Vue 3 compilation
    "@wxt-dev/auto-icons",      // Icon generation from logo
  ],
  manifest: {
    name: "Lumin Time",
    homepage_url: "https://github.com/nuttycc/LuminTime",
    permissions: ["tabs", "storage", "webNavigation", "idle", "alarms"],
  },
  vite: () => ({
    plugins: [tailwindcss()],    // Tailwind CSS processing
  }),
});
```

### Package Info

Version and metadata from `package.json`:

```json
{
  "name": "lumin-time",
  "version": "0.1.18",          // Extension version
  "description": "A privacy-first browser extension that automatically tracks your online activity.",
  "homepage": "https://github.com/nuttycc/LuminTime"
}
```

WXT reads `version` from `package.json` and injects it into `manifest.json`.

## Browser-Specific Builds

### Chrome (Manifest V3)

Generated manifest:

```json
{
  "manifest_version": 3,
  "name": "Lumin Time",
  "version": "0.1.18",
  "permissions": ["tabs", "storage", "webNavigation", "idle", "alarms"],
  "background": {
    "service_worker": "background.js",
    "type": "module"
  },
  "action": {
    "default_popup": "popup.html"
  }
}
```

**Supports:**
- Chrome 88+
- Edge 88+
- Brave
- Opera

### Firefox (Manifest V2)

Generated manifest:

```json
{
  "manifest_version": 2,
  "name": "Lumin Time",
  "version": "0.1.18",
  "permissions": ["tabs", "storage", "webNavigation", "idle", "alarms"],
  "background": {
    "scripts": ["background.js"]
  },
  "browser_action": {
    "default_popup": "popup.html"
  },
  "browser_specific_settings": {
    "gecko": {
      "strict_min_version": "109.0"
    }
  }
}
```

**Supports:**
- Firefox 109+
- Firefox Developer Edition

<Note>
  WXT automatically handles API differences between browsers. For example, `chrome.tabs` becomes `browser.tabs` in Firefox builds.
</Note>

## Build Optimization

### Code Splitting

WXT splits code by entry point:

```
background.js    # Background service worker
popup.html       # Popup UI entry
  ├─ popup.js    # Vue app bundle
  └─ popup.css   # Tailwind styles
```

### Tree Shaking

Unused code is automatically removed:

```typescript
// If you import:
import { formatDate } from '@/utils/date';

// Only formatDate is bundled, other utils are excluded
```

### Asset Optimization

- **Images**: Compressed and optimized
- **Icons**: Generated in multiple sizes (16x16, 32x32, 48x48, 128x128)
- **CSS**: Minified, unused styles removed by Tailwind
- **JS**: Minified with Terser

### Bundle Analysis

Analyze bundle size:

```bash
bun run build --analyze
```

Opens interactive bundle visualization in browser.

## Version Management

### Bumping Version

Update version in `package.json`:

```json
{
  "version": "0.2.0"  // Increment here
}
```

Then rebuild:

```bash
bun run build
bun run zip
```

### Semantic Versioning

Follow [semver](https://semver.org/):

- **Major** (1.0.0 → 2.0.0): Breaking changes
- **Minor** (0.1.0 → 0.2.0): New features, backward compatible
- **Patch** (0.1.0 → 0.1.1): Bug fixes

## Publishing

### Chrome Web Store

<Steps>
  <Step title="Create Developer Account">
    Register at [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole).
    
    One-time $5 registration fee.
  </Step>
  
  <Step title="Upload Package">
    1. Click "New Item"
    2. Upload `.output/lumin-time-0.1.18-chrome.zip`
    3. Fill in store listing details
  </Step>
  
  <Step title="Submit for Review">
    Review typically takes 1-3 business days.
  </Step>
</Steps>

### Firefox Add-ons (AMO)

<Steps>
  <Step title="Create Developer Account">
    Register at [Firefox Add-on Developer Hub](https://addons.mozilla.org/developers/).
    
    Free, no registration fee.
  </Step>
  
  <Step title="Submit Extension">
    1. Click "Submit a New Add-on"
    2. Upload `.output/lumin-time-0.1.18-firefox.zip`
    3. Choose distribution channel (public or self-hosted)
  </Step>
  
  <Step title="Automated Review">
    Most extensions auto-approved within minutes.
    
    Manual review required if code is obfuscated or uses broad permissions.
  </Step>
</Steps>

### Edge Add-ons

<Steps>
  <Step title="Use Chrome Package">
    Edge uses the same package as Chrome (Manifest V3).
  </Step>
  
  <Step title="Register as Partner">
    Sign up at [Microsoft Partner Center](https://partner.microsoft.com/).
  </Step>
  
  <Step title="Submit Extension">
    Upload the Chrome ZIP package to Edge Add-ons store.
  </Step>
</Steps>

## Automated Release

Automate builds with GitHub Actions:

```yaml
# .github/workflows/release.yml
name: Release
on:
  push:
    tags:
      - 'v*'  # Trigger on version tags

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: oven-sh/setup-bun@v1
      
      - name: Install dependencies
        run: bun install
      
      - name: Run tests
        run: bun run test
      
      - name: Build extension
        run: bun run zip
      
      - name: Create GitHub Release
        uses: softprops/action-gh-release@v1
        with:
          files: |
            .output/*.zip
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**Usage:**

```bash
git tag v0.2.0
git push origin v0.2.0
# GitHub Actions builds and creates release with ZIP files
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with 'Module not found'">
    **Solution:**
    1. Run `bun install` to ensure all dependencies are installed
    2. Check import paths (use `@/` alias for `src/`)
    3. Verify TypeScript types with `bun run compile`
  </Accordion>
  
  <Accordion title="Built extension doesn't work">
    **Solution:**
    1. Check browser console for errors
    2. Inspect `manifest.json` in output directory
    3. Verify all required files are present in build
    4. Test dev build first: `bun run dev`
  </Accordion>
  
  <Accordion title="ZIP command not found">
    **Solution:**
    WXT uses `zip` utility. Install it:
    
    ```bash
    # macOS (via Homebrew)
    brew install zip
    
    # Ubuntu/Debian
    sudo apt install zip
    
    # Windows (use WSL or Git Bash)
    ```
  </Accordion>
  
  <Accordion title="Bundle size too large">
    **Solution:**
    1. Run `bun run build --analyze` to identify large dependencies
    2. Replace heavy libraries with lighter alternatives (es-toolkit vs lodash)
    3. Use dynamic imports for rarely-used code
    4. Remove unused assets and dependencies
  </Accordion>
</AccordionGroup>
