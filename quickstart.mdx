# Quickstart Guide

This guide will help you get started with LuminTime quickly. Within minutes, you'll have the extension installed and tracking your browsing activity.

## Installation

<Steps>
  <Step title="Install the Extension">
    Install LuminTime from the [Chrome Web Store](https://chromewebstore.google.com/detail/lumin-time/elfkjlledpjmjehekdklejboanfbffak):
    
    1. Click **"Add to Chrome"**
    2. Confirm by clicking **"Add extension"**
    3. The LuminTime icon will appear in your browser toolbar
    
    <Info>
      The extension begins tracking automatically after installation—no setup required.
    </Info>
  </Step>
  
  <Step title="Start Browsing">
    Simply browse the web as you normally would. LuminTime runs silently in the background, tracking:
    
    - Time spent on each website
    - Active tab changes
    - Window focus events
    - Idle time detection
    
    <Note>
      LuminTime only tracks when you're actively using a tab. If your computer goes idle or you switch to another application, tracking pauses automatically.
    </Note>
  </Step>
  
  <Step title="View Your Analytics">
    Click the LuminTime icon in your browser toolbar to open the popup dashboard. You'll see:
    
    - Total time tracked today
    - Top websites by duration
    - Hourly activity distribution
    - Daily usage trends
  </Step>
</Steps>

## Understanding the Dashboard

The LuminTime popup provides several views to explore your browsing data:

### Home View

The default view shows your current day's activity:

- **Total Time Today** – Cumulative time spent browsing
- **Top Sites** – Websites ranked by time spent
- **Hourly Breakdown** – Visual timeline of your activity

### History View

Access detailed historical data:

- Browse activity by date
- See time spent per site for any day
- Identify usage patterns over time

### Insights View

Get analytics on your browsing habits:

- Daily averages and trends
- Most visited domains
- Peak activity hours

### Site Details

Click any website to see granular details:

- Individual page visits
- Time spent per URL
- Navigation patterns within a domain

## Key Concepts

### Session Tracking

LuminTime tracks browsing sessions using the Browser Tabs API:

```typescript
// Extension monitors these events:
- tabs.onActivated   // When you switch tabs
- tabs.onUpdated     // When a page loads or URL changes
- windows.onFocusChanged  // When you switch browser windows
- idle.onStateChanged     // When you step away from the computer
```

A "session" represents continuous time spent on a specific page. Sessions end when you:
- Switch to a different tab
- Navigate to a new page
- Close the tab
- Go idle (after system idle timeout)

### Midnight Boundaries

LuminTime automatically splits sessions at midnight (00:00) to ensure accurate daily analytics. If you're browsing at 11:45 PM and continue past midnight:

- Session 1: 11:45 PM - 11:59:59 PM (counted for Day 1)
- Session 2: 12:00 AM - [end time] (counted for Day 2)

This ensures your daily totals are always accurate without manual intervention.

### Local-First Storage

All data is stored in IndexedDB with three layers:

1. **Raw History Logs** – Granular session records with start/end times
2. **Site Aggregates** – Daily totals per domain
3. **Page Details** – URL-specific statistics

<Note>
  LuminTime uses [Dexie.js](https://dexie.dev/), a powerful IndexedDB wrapper, for efficient local storage. No data ever leaves your device.
</Note>

## Privacy & Data

### What Data is Tracked?

LuminTime records:
- ✅ Website domains (e.g., `github.com`)
- ✅ Full URLs (e.g., `github.com/nuttycc/LuminTime`)
- ✅ Start and end timestamps
- ✅ Duration spent on each page

LuminTime does NOT record:
- ❌ Page content or text
- ❌ Passwords or form inputs
- ❌ Cookies or authentication tokens
- ❌ Screenshots or visual data

### Where is Data Stored?

All data lives in your browser's IndexedDB:

```bash
# Data location (Chrome on Linux)
~/.config/google-chrome/Default/IndexedDB/chrome-extension_[id]/

# Data location (Chrome on macOS)
~/Library/Application Support/Google/Chrome/Default/IndexedDB/chrome-extension_[id]/

# Data location (Chrome on Windows)
%LOCALAPPDATA%\Google\Chrome\User Data\Default\IndexedDB\chrome-extension_[id]\
```

<Info>
  Your data is isolated to your browser profile. If you use multiple Chrome profiles or browsers, each has its own separate LuminTime database.
</Info>

### Is My Data Synced?

No. LuminTime deliberately does not use Chrome Sync or any cloud service. Your data stays on your device permanently. This means:

- ✅ Complete privacy—no one can access your data
- ⚠️ Data is not synced across devices
- ⚠️ Uninstalling the extension deletes all data

## Tips for Best Results

### Keep the Extension Enabled

LuminTime tracks continuously while enabled. If you disable it, tracking stops and you'll have gaps in your data.

### Let It Run for a Few Days

Meaningful insights emerge after a few days of tracking. Daily trends and weekly patterns become visible over time.

### Check Insights Regularly

The Insights view helps identify browsing habits. Review it weekly to understand how you spend time online.

### Use Multiple Views

Different views serve different purposes:
- **Home** – Quick daily overview
- **History** – Detailed past exploration
- **Insights** – Pattern recognition and trends
- **Site Details** – Granular per-page analysis

## Troubleshooting

### Extension Not Tracking

If you're not seeing any data:

1. Verify the extension is enabled in `chrome://extensions/`
2. Check that the extension has necessary permissions
3. Click the icon to open the popup—tracking starts immediately
4. Browse for a few minutes, then check again

### Missing Time Data

If some sessions aren't recorded:

1. Ensure you're not in Incognito mode (extensions are disabled by default)
2. Check if the extension has permission to run in Incognito (if desired)
3. Verify your system idle timeout isn't too aggressive

### Data Suddenly Gone

If your data disappears:

1. Check if you accidentally cleared browser data (Settings → Privacy)
2. Verify you're using the same browser profile
3. Ensure the extension wasn't reinstalled (reinstalling wipes data)

<Note>
  Always be cautious when clearing browser data. IndexedDB is included in "Cookies and site data" and will be deleted if you clear it.
</Note>
